JavaScript Type Conversion & Operations

Type Conversion Examples

let value = Number(null);
Result: null → 0 (number)
When converting null to a number, JavaScript returns 0.
let value = Boolean(1);
Result: 1 → true (boolean)
When converting numbers to boolean: 0 → false, any other number → true.
let value = String(33);
Result: 33 → "33" (string)
Converting a number to a string results in the string representation of that number.

Arithmetic Operations

console.log(1 + 2 + "2");
Result: "32" (string)
Operations are performed left to right:
1. 1 + 2 = 3 (numeric addition)
2. 3 + "2" = "32" (string concatenation)
console.log(((3 + 4) * 5) % 3);
Result: 2 (number)
Step by step calculation:
1. (3 + 4) = 7
2. 7 * 5 = 35
3. 35 % 3 = 2 (remainder when 35 is divided by 3)
console.log(+true); // Output: 1
console.log(+""); // Output: 0
Unary plus converts values to numbers
The unary plus operator (+) attempts to convert the operand to a number:
- +true converts boolean true to number 1
- +"" converts empty string to number 0

JavaScript Type Conversion Rules

Original Value Converted to Number Converted to Boolean Converted to String
null 0 false "null"
undefined NaN false "undefined"
true 1 true "true"
false 0 false "false"
"" (empty string) 0 false ""
"123" 123 true "123"
"123abc" NaN true "123abc"
0 0 false "0"

Key Takeaways

  • JavaScript performs implicit type conversion (coercion) in operations involving different types
  • The + operator favors string concatenation when one operand is a string
  • Other arithmetic operators (- * / %) convert operands to numbers
  • Unary plus (+) can be used to explicitly convert values to numbers
  • Understanding type conversion is essential to avoid unexpected behavior in JavaScript
